# 2026-03-09T12:00:00Z — DNS/mDNS hostname resolution for `WSPR_MULTICAST_ADDR`

## Problem

`WSPR_MULTICAST_ADDR` only accepted bare IP literals.  Users running Avahi on
their LAN register multicast receivers under `.local` mDNS names
(e.g. `radio.local`); these could not be used directly.

## Solution

Add a `resolve_host(s: &str) -> Result<IpAddr>` helper in `src/config.rs` that:

1. **Fast path** — tries `s.parse::<IpAddr>()` directly; no syscall for plain
   IP literals.
2. **Slow path** — calls the OS `getaddrinfo(3)` resolver via
   `std::net::ToSocketAddrs`.  On Linux with `nss-mdns` installed this
   transparently resolves `.local` Avahi names; no extra crate required.
