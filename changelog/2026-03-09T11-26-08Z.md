# 2026-03-09T11:26:08Z — Quiet mode for systemd / non-TTY operation

## Problem

Progress bars and ANSI escape codes written to stderr are inappropriate (and
actively harmful) when running under systemd — journald receives raw escape
sequences and the bar rendering corrupts log output.

## Solution

Auto-detect quiet mode: if `stderr` is not a TTY (always the case under
systemd) **or** if `WSPR_QUIET=1` is set, suppress bars and write clean
plain-text logs.

## Changes

### `src/main.rs`

**Replaced** `MultiProgressMakeWriter` / `MultiProgressEventWriter` with a
unified `AppMakeWriter` / `AppWriter` enum that handles both modes in a
single type:

```
AppWriter::Tty   — buffers event, flushes via MultiProgress::println (as before)
AppWriter::Quiet — writes directly to std::io::stderr()
```

**Quiet mode detection** (`main`):
```rust
let quiet = !std::io::stderr().is_terminal()
    || std::env::var("WSPR_QUIET").map(|v| v == "1").unwrap_or(false);
```

**`MultiProgress`** is created with `ProgressDrawTarget::hidden()` in quiet
mode — all `ProgressBar` calls in `buffer_task` become no-ops with no
rendering overhead.  No changes to `buffer_task` were required.

**`with_ansi(!quiet)`** added to the tracing format layer — strips ANSI
escape codes from log lines when writing to journald.

### `wsprrs.service`

New sample systemd unit file.  Configures:
- `User=wsprrs` (dedicated system user)
- `EnvironmentFile=/etc/wsprrs/.env`
- `WorkingDirectory=/etc/wsprrs` (wisdom file lands here by default)
- `StandardOutput/StandardError=journal`
- `Restart=on-failure`
