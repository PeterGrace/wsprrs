# 2026-03-09T10:42:03Z ‚Äî ALL_WSPR.TXT parsing + FFTW wisdom management

## Summary

Switched spot ingestion from `wsprd` stdout to `ALL_WSPR.TXT` for richer
decode-quality metrics.  Added per-decode isolated working directories to
prevent concurrent collision.  Added FFTW wisdom file propagation to avoid
redundant FFT planning on subsequent runs.

## Changes

### `src/spot.rs`

Replaced `jitter: u32` (never populated) with the full ALL_WSPR.TXT field
set: `sync_quality: f32`, `npass: u8`, `osd_pass: u8`, `nhardmin: i32`,
`decode_cycles: u32` (now populated), `candidates: u32`, `nfano: i32`.

### `src/config.rs`

Added `wisdom_file: PathBuf` (env `WSPR_WISDOM_FILE`, default
`wspr_wisdom.dat` in CWD).

### `src/decode.rs`

**`write_wav_file`** ‚Äî signature changed from `(window, temp_dir: &str) ->
Result<NamedTempFile>` to `(window, path: &Path) -> Result<()>`.  Uses an
atomic temp-then-persist pattern internally.

**`copy_wisdom_in(wisdom_path, work_dir)`** ‚Äî copies `wspr_wisdom.dat` into
the per-decode temp subdirectory before `wsprd` runs.  Silent no-op if the
file does not yet exist (expected on first run).

**`copy_wisdom_out(work_dir, wisdom_path)`** ‚Äî after `wsprd` exits, copies
the newly generated `wspr_wisdom.dat` back to the configured path if one is
not already present there.  Uses write-then-atomic-rename; concurrent calls
from parallel decode tasks are safe because FFTW wisdom is deterministic for
a given CPU (all tasks write identical content).

**`run_wsprd`** ‚Äî now accepts `work_dir: &Path` and runs `wsprd` with
`Command::current_dir(work_dir)`.  Reads `ALL_WSPR.TXT` from `work_dir`
after the process exits; returns an empty `Vec` if the file was not created
(no spots decoded).

**`is_grid_square`** ‚Äî extended from 4-char only to 4- or 6-char Maidenhead
locators; subsquare pair is matched case-insensitively (`A-X` / `a-x`).

**`parse_all_wspr_txt` / `parse_all_wspr_line`** ‚Äî new positional parser for
the 16‚Äì17 column ALL_WSPR.TXT format.  Grid presence detected by
`is_grid_square(f[6])`; `base` index adjusts all subsequent fields.
`nbadcrc` column is read but discarded (always 0 for a valid CRC decode).

Removed all stdout-based parsing (`parse_wsprd_output`, `parse_spot_line`,
`parse_new_format`, `parse_old_format`).

### `src/main.rs` ‚Äî `decode_window`

Restructured to:
1. `spawn_blocking`: create `TempDir` in `cfg.temp_dir`; write WAV to
   `{work_dir}/wspr.wav`.
2. `copy_wisdom_in` ‚Äî copy wisdom into work dir.
3. `run_wsprd` ‚Äî subprocess in work dir; reads ALL_WSPR.TXT.
4. `copy_wisdom_out` ‚Äî propagate wisdom if newly generated.
5. Drop `TempDir` ‚Äî entire subdirectory cleaned up.

## FFTW wisdom flow

```
First run:                       Subsequent runs:
  copy_wisdom_in ‚Üí not found       copy_wisdom_in ‚Üí copies wisdom in
  wsprd runs, creates wisdom       wsprd runs (skips FFT planning)
  copy_wisdom_out ‚Üí saves to CWD   copy_wisdom_out ‚Üí already exists, no-op
```
