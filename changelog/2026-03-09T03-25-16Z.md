# 2026-03-09T03:25:16Z — Fix: seal window on BufferOverflow instead of discarding

## Problem

Every WSPR window was being silently discarded at the end of the 2-minute
capture period with the error `IQ buffer overflow for SSRC 0x…`.

### Root cause

`AudioWindow::ingest()` returns `WsprError::BufferOverflow` when a new RTP
packet would extend past the pre-allocated buffer.  This is **expected normal
behavior** near the end of the 116-second capture window:

1. A window opened at wall-clock second 38 of its cycle sets
   `start_offset = 38 × 12 000 = 456 000 samples`.
2. By second ~116, the RTP write cursor sits at
   `456 000 + 77 × 12 000 ≈ 1 380 000 samples` with only ~12 000 samples
   of headroom.
3. Any standard 80 ms RTP packet (960 samples) causes `offset + n > capacity`,
   triggering the error.

The old error handler called `windows.remove(&ssrc)` — silently throwing away
a fully-captured window without ever sending it to `decode_task`.

Windows that first appeared very late in the cycle (within the last second)
also opened immediately and then overflowed on their first packet for the same
reason.

## Fix (`src/main.rs`)

Replaced the single catch-all `Err(e)` handler in `buffer_task` with two
distinct arms:

```rust
Err(WsprError::BufferOverflow { .. }) => {
    // Buffer full — seal for decode, not discard.
    if let Some((window, pb)) = windows.remove(&ssrc) {
        pb.finish_with_message("sealing");
        tracing::info!(..., "buffer full: sealing audio window for decode");
        win_tx.send((window, cfg.clone())).await …;
    }
}
Err(e) => {
    // Genuine unexpected error — discard.
    tracing::error!(…);
    windows.remove(&ssrc);
}
```

`BufferOverflow` is now treated identically to the 1-second expiry tick:
the window is forwarded to `decode_task` for `wsprd` processing.

Added `use error::WsprError;` import to enable pattern matching on the
variant.

## Impact

- All captured WSPR windows are now decoded instead of discarded.
- No change to the public API or any data structures.
- All 37 unit tests pass; clippy clean with `-D warnings`.
