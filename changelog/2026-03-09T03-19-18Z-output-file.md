# NDJSON Spot Output File — 2026-03-09T03:19:18Z

## Feature

Add optional `WSPR_OUTPUT_FILE` environment variable.  When set, each decoded
WSPR spot is appended to the named file as a single JSON line (NDJSON /
JSON-Lines format) in addition to the existing `tracing::info!` output.

## Behaviour

- File is opened once at `decode_task` startup in **append + create** mode, so
  repeated runs accumulate spots rather than overwriting previous results.
- Each spot is flushed immediately after writing, so the file is safe to `tail
  -f` or parse incrementally.
- If the file cannot be opened, a warning is logged and spots continue to
  appear only in the tracing log — the decode pipeline is not interrupted.
- When unset (default), behaviour is identical to before this change.

## Configuration

```
# .env
WSPR_OUTPUT_FILE=/var/log/wsprrs/spots.ndjson
```

## Output Format

One JSON object per line:

```json
{"time_utc":"0300","snr_db":-18,"dt_sec":0.3,"freq_hz":14097044.0,"message":"K1ABC FN42 33","grid":"FN42","power_dbm":33,"drift":0,"decode_cycles":35,"jitter":3}
```

## Files Changed

- **`src/config.rs`** — added `output_file: Option<String>` field; parses
  `WSPR_OUTPUT_FILE`
- **`src/main.rs`** — `decode_task` now accepts `cfg: Arc<Config>`; opens
  `BufWriter<tokio::fs::File>` if `output_file` is set; added `write_spot()`
  helper that appends one JSON line and flushes per spot
